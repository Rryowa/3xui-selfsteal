# Testing Guide

This guide outlines how to verify that the 3x-ui panel, Nginx decoy, and VLESS xHTTP connections are configured correctly and active.

## Automated Testing

Run the built-in integration tests from the root directory:
```bash
make test
```

This script checks:
1. Docker containers are active (`nginx-selfsteal`, `3xui_app`).
2. The SQLite database at `/opt/3x-ui/db/x-ui.db` is configured with `xhttp-inbound` and client `user-xhttp`.
3. The Unix socket file `/dev/shm/nginx-xhttp.socket` exists.
4. Nginx correctly terminates TLS and reverse proxies the xHTTP stream to the socket, returning HTTP/2 200.
5. The 3x-ui panel is reachable over port 8443 via HTTPS.

---

## Manual Testing

### 1. Verify VLESS Link Parameters
Ensure client configuration links generated by the panel have the following parameters:
* **Protocol**: `vless`
* **Address**: Your decoy domain (`your-domain.com`)
* **Port**: `443`
* **Transport/Network**: `xhttp`
* **Path**: `/api/v1/assets/logo.png`
* **Security**: `TLS` (Nginx handles TLS termination, client connects via SSL/TLS)

### 2. Verify Decoy Server
Visit `https://your-domain.com` in a browser or curl it. You should see a clean, professional decoy site (e.g. Speedtest, YouTube clone) rather than Nginx default page.

### 3. Check Socket and Container Connection
Ensure the shared socket file can be read and written to:
```bash
docker exec -it nginx-selfsteal ls -lh /dev/shm/nginx-xhttp.socket
docker exec -it 3xui_app ls -lh /dev/shm/nginx-xhttp.socket
```
